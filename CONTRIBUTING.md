# Contributing

Thanks for your interest in contributing to this repository. A few quick guidelines:

- Issues: Open issues for bugs or feature requests. Provide a minimal reproduction if possible.
- Pull Requests: Fork the repository, create a branch for your change, and open a PR against `master`.
- Tests: If you add functionality, include a small verification (e.g., a script or unit test). CI runs `verify_install.py`.
- Style: Follow existing code style. Keep changes focused and include a short description in the PR.
- Communication: Use PR comments for discussion and link to any related issues.

We appreciate clear commit messages and small incremental PRs.
